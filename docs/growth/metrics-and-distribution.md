# Beacon — Growth Metrics & Distribution Playbook

Date: 2026-07-05. Sources: nestymee (AI UGC $2–4/video; TikTok ~$0 CAC), James
Ebringer (TikTok account warm-up), Marc Lou (31 Principles of a Viral Product),
Frederick James (Apple "Admaxxing": $0→$1k MRR in 25 days via Apple Search Ads).

## Funnel diagnostics (the four failure modes)

| Symptom | Diagnosis | Lever |
|---|---|---|
| Low downloads | Distribution weak | TikTok organic + Apple Search Ads |
| Low trial starts | Onboarding confusing | Shorten time-to-first-itinerary |
| No conversions | Paywall bad | Paywall copy/placement (Marc Lou principles) |
| Churn | App loop isn't as promised | Itinerary quality, repeat-trip value |

## Metrics

**North star: weekly itineraries generated by distinct users.** Proxy for the
promise being delivered; everything else feeds or monetizes it.

### Maximize

| Metric | Stage | Target (first 90 days) |
|---|---|---|
| Downloads / week | Distribution | 500+/wk by day 60 |
| TikTok views / week | Distribution | 100K/wk (nestymee benchmark) |
| Hook hold rate (first 3–5 s) | Content QC | ≥60% before promoting a video (nestymee gate) |
| Install → first itinerary, same session | Activation | ≥50% |
| Onboarding completion rate | Activation | ≥80% |
| Paywall view → trial start | Conversion | ≥8% |
| Trial → paid | Conversion | ≥35% |
| D1 / D7 / D30 retention | Loop | 35% / 15% / 8% |
| MRR | Business | $1k by ~day 25–30 of paid spend (Admaxxing target) |

### Minimize

| Metric | Stage | Target |
|---|---|---|
| Blended CAC | Distribution | ~$0 organic; paid CPA < 1-month sub price |
| Apple Ads cost-per-install | Distribution | profitable within week 1 or kill the keyword |
| Time-to-first-itinerary | Activation | < 2 min from open |
| Onboarding drop-off per step | Activation | no step loses > 15% |
| Trial cancellation before conversion | Conversion | < 50% |
| Monthly sub churn | Loop | < 10% |
| Refund rate | Loop | < 2% |

Instrumentation: RevenueCat already covers trial/conversion/churn/MRR. Nothing
covers downloads→activation funnel — need App Store Connect analytics (downloads,
conversion rate) + lightweight event tracking (onboarding step completed, itinerary
generated, paywall viewed). PostHog free tier or plain Supabase events table.

## Distribution plan

### 1. TikTok organic (primary, ~$0 CAC)

Account warm-up (Ebringer) — before posting anything:
- Day 1–2: create account, post nothing, no bio link. Scroll FYP 30 min in the
  travel niche, like 15–20 videos, leave 5–7 real comments (not emojis), follow
  8–10 similar accounts.
- Continue until FYP is ~70% niche content, then start posting. Add bio link later.

Content ladder (nestymee) — validate cheap before spending:
1. **Trendwatch (free):** use Claude to analyze travel-app/travel-content accounts
   under 500k followers; extract hook formats that perform relative to follower
   count; mine comments for audience language.
2. **Carousels (cents):** test hooks/messages as photo carousels first. Underperforms
   as carousel → will flop as video at 5× cost.
3. **AI UGC video ($2–4):** consistent AI persona (Higgsfield Soul), viral-style
   first frame (Nano Banana Pro / GPT Image 2), Kling 3 for simple shots, Seedance 2
   for speech/detail. Structure: 3–5 s hook + 5–7 s app demo assembled in Remotion
   from existing screen recordings (demo is free, only the hook generation costs).
4. **Distribution (Postiz):** Claude drives the Postiz API to spin hook variants and
   schedule one video as a week of differently-framed posts across accounts.

Account health: never post identical content back-to-back (shadowban risk), human
posting cadence, coherent voice per account, optimize for saves + watch-time.
Manual QC gate: only promote videos holding ≥60% through first 3–5 s.

### 2. Apple Search Ads (paid, "Admaxxing")

- Small daily budget (~$5–10/day), exact-match, high-intent keywords: "travel
  itinerary planner", "trip planner ai", competitor names (Wanderlog, TripIt).
- Rule: each keyword must show install→trial economics beating 1-month sub price
  within week 1, else kill. Target $1k MRR in ~25 days of profitable spend.
- Needs the App Store product page done first (screenshots currently ZERO — see
  app-store-submission state). ASA traffic converts on screenshots + subtitle.

### 3. Product page & paywall (Marc Lou principles, applied)

- Numbers over adjectives: "A 5-day Tokyo itinerary in 60 seconds", not "plan trips fast".
- Sell outcome, not features: time saved, no research spiral, no 40 open tabs.
- One CTA per screen, specific verb: "Build my itinerary", not "Get started".
- Show product first: screenshots/demo video lead with a finished itinerary.
- Ten-word description test for subtitle + first screenshot caption.
- Emotional, fifth-grade headline; test ~5, keep what people recall.
- Hard paywall stays (aligned with his "ask for payment early"); testimonials on the
  paywall once first reviews exist ("collect proof before traffic").
- Deliberate deviations: keeping free itinerary-before-paywall taste and subscription
  model — standard for travel apps; his no-free/no-subs advice is for web one-shots.

## Weekly loop

Post daily (after warm-up) → check the funnel Monday → the four diagnostics point at
the one lever to pull that week. Don't fix the paywall when downloads are the
bottleneck, and vice versa.
